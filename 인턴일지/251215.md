## 2. 업무 진행상황


<aside>

**오전 (10:00-12:00)**

**Stripe 대시보드**

- 업그레이드 비중 그래프
    - 지표 정의
        1. **전환 효율 (Upgrade Rate)**
            - 업그레이드가 가능한 사람들 중 실제로 얼마나 잘 전환시키고 있는가?
            - 예: Starter → Creator 업그레이드 고객 수 / 월 초 기준 Starter 플랜 고객 수
        2. **비즈니스 침투율 (Upgrade Penetration)**
            - 전체 고객 풀 대비 업그레이드가 어느 정도 규모로 발생하고 있는가?
            - 예: Starter → Creator 업그레이드 고객 수 / 월 초 기준 전체 활성 구독자 수
        3. **두 지표 모두 분모는 월 초 기준을 사용한다.**
            - 월말 기준을 사용할 경우, 이미 업그레이드한 고객이 분모에서 제외되어 업그레이드 비율이 왜곡될 수 있다.
        4. **확률적 관점에서 가장 정확한 정의는**
            - 해당 기간 동안 업그레이드가 가능했던 모든 고객을 분모에 포함하는 것이다.
                - 기간 시작 시점의 하위 플랜 고객
                - 기간 중 신규 유입된 하위 플랜 고객
                - 기간 중 재활성화된 하위 플랜 고객
                    
                    을 모두 포함한 ‘risk set’을 기준으로 계산한다.
                    
        5. **다만 KPI 지표로 활용할 경우에는**
            - 지표의 일관성과 비교 가능성을 위해 분모를 ‘월 초 기준 활성 고객’으로 단순화하여 사용
- 이탈률 그래프
    - Stripe에서 제공하는 이탈률 그래프는 30일 rolling 그래프
    - 분자 = 30일간 이탈한 사람 수
    - 분모 = 30일 전 활성 구독자 수 + 30일간 신규 구독자 + 30일간 재활성화 구독자
        - 지난 30일 동안 한 번이라도 유료였던 고객 전체

**오후 (13:00-16:00)**

**Stripe 지표 정의**

- **업그레이드 비중 그래프**
    1. **전환 효율 (Upgrade Rate) → 해당 방식으로 구현**
        - 업그레이드가 **가능했던 고객 중** 실제로 업그레이드한 비율
        - 분자: 해당 기간 Starter → Creator 업그레이드 고객 수
        - 분모: **월 초 기준** Starter 플랜 활성 고객 수
    2. **비즈니스 침투율 (Upgrade Penetration)**
        - 전체 고객 풀 대비 업그레이드 발생 규모
        - 분자: 해당 기간 Starter → Creator 업그레이드 고객 수
        - 분모: **월 초 기준** 전체 활성 구독자 수
- **이탈률 그래프**
    1. **확률 기반 이탈률 (Rolling Churn Rate) → 해당 방식으로 구현**
        - 최근 30일 동안 해당 플랜을 보유했던 고객 중 이탈한 비율
        - 분자: 최근 30일간 해당 플랜에서 이탈한 고객 수
        - 분모: 최근 30일 동안 해당 플랜을 한 번이라도 보유했던 고객 전체 (risk set)
            - 30일 전 활성화 고객+신규+재구독
    2. **스냅샷 기반 이탈 비중 (Snapshot Churn Ratio)**
        - 현재 해당 플랜 고객 규모 대비 이탈 발생 비중
        - 분자: 해당 기간 동안 **해당 플랜에서 이탈한 고객 수**
        - 분모: **월말 기준 해당 플랜 활성 구독자 수**

**Stripe 대시보드 구현**

- 업그레이드 전환율 구현중
