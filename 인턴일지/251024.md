## 2. 업무 진행상황

<aside>

**오전 (10:00-12:00)**

**대시보드 쿼리 작성**

- 여러 구독 보유하고 있는 사용자 확인

**오후 (13:00-16:00)**

**대시보드 구성**

- 개요-주의사항에 들어갈 차트
- 사용자 탭-상세 차트

**회의**

더빙, 스튜디오 - free 포함인지 확인

**데일리 스크럼**

**기존 내용**

- 탈퇴 유저를 leave + delete 유저로 집계
- 총 사용자 수를 집계하여 사용자 추이 그래프에 표시
- 신규 구독자 수 집계 추가

**수정 사항**

- 탈퇴 유저를 leave, 사용자 삭제 유저를 delete로 각각 집계
    - leave 유저는 30일 후 delete 상태로 전환됨
    - 탈퇴 조회는 update_date 기준으로 이루어지며, delete 상태로 변경될 때도 해당 일자가 갱신됨
    - 당일 조회 시 leave로 집계된 사용자가 30일 후 delete로 다시 집계되어 중복 반영될 우려가 있음
- 총 사용자 수는 당일 기준 지표로만 사용하며, 사용자 추이 그래프에는 신규·탈퇴·삭제 지표만 반영
    - 사용자 상태(status)는 실시간으로 갱신되므로, 과거 시점에 normal이었던 사용자라도 현재 leave 처리된 경우 조회 결과에서 제외됨
    - 당일 지표는 정상적으로 조회 가능함
- 신규 구독자 집계 보류
    - event_type = created 데이터는 모두 status = incomplete(미구독)로 확인됨
    - first_start_date를 활용한 집계 시 기준 정의가 불명확하여 일시적으로 보류함
- 대시보드 UI
    - 기존 탭 : 개요, 사용자, 구독자, 크레딧 → 변경 탭 : 개요, 구독자, 크레딧
    - 상단에 노출되는 카드 개수를 페이지 별로 다르게 하는게 더 좋아 보인다
        - 사유: 각 항목마다 중요한 지표 개수가 다르기 때문
    - 기간 필터링은 최대 3개월
